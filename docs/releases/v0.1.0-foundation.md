# Release Notes — Foundation v0.1.0-foundation

**Tag:** [`v0.1.0-foundation`](https://github.com/OWNER/nexus-anime/releases/tag/v0.1.0-foundation)
**Date:** 2026-06-27
**Milestones:** M0 (Repository scaffold) + M1 (Design system in code)

This release finalizes the Project Foundation milestone. The repository is
stable, fully documented, and validated end-to-end — ready for UI development
to begin on short-lived feature branches.

---

## Completed work

### M0 — Repository scaffold

- Turborepo + pnpm workspace monorepo (`apps/*`, `packages/*`, `tooling/`, `docs/`).
- Next.js 16 (App Router) + React 19 deployable web app (`apps/web`).
- Shared packages: `@nexus/ui` (design system), `@nexus/db` (Drizzle/Neon
  placeholder), `@nexus/cache` (Upstash Redis placeholder),
  `@nexus/eslint-config` (shared ESLint 9 flat config with package-boundary
  enforcement).
- Local dev infrastructure: Docker Compose (Postgres 16, Redis 7, Mailpit) +
  root compose that builds the production `web` image.
- Seed scripts for admin, anime, and catalog data.
- CI workflow (lint, typecheck, test, build, format:check) + Release workflow.
- Issue/PR templates, CODEOWNERS, branch protection conventions.
- Repository constitution: `docs/REPOSITORY-DESIGN.md` (all 28 deliverables).
- Master roadmap, milestone specs, and ADR index.

### M1 — Design system in code

- `@nexus/ui` component library scaffold (shadcn/ui-based primitives).
- Tailwind CSS v4 theme tokens (Void/Aether/Nova palettes, semantic surfaces,
  typography, spacing, radius, shadows, blur, motion, z-index) registered as
  Tailwind utilities via `@theme`.
- `globals.css` entry point wiring `tailwindcss`, `tw-animate-css`, and the
  `@nexus/ui/tailwind` token layer.
- `/api/health` route for container health checks.
- Env-validation entrypoint (`apps/web/scripts/entrypoint.sh`).

### Release audit fixes (this pass)

- Added root `eslint.config.js` + `tsconfig.json` — `pnpm lint` now resolves a
  config from the repo root (previously failed under ESLint 9 flat config).
- Fixed `apps/web/Dockerfile`: idempotent non-root `useradd`, and copy
  `pnpm-lock.yaml` into the deps stage so `--frozen-lockfile` builds succeed.
- Promoted `tw-animate-css` to a production dependency of `apps/web` (the
  production Docker build could not resolve it from `@nexus/ui`'s devDependencies).
- Ignored `.claude/` and `.superpowers/` in `.prettierignore` (tooling, not source).

## Known limitations

- **No application features yet.** This is a foundation release: catalog, auth,
  profiles, payments, and video are not implemented.
- `@nexus/db` and `@nexus/cache` are placeholders — no schema, migrations, or
  Redis client code.
- No real tests yet — package `test` scripts are stubs (`echo "no tests"`).
  Test harness (Vitest) is not wired.
- No auth/session — middleware and route guards are not present.
- The design system has a token layer and a Button primitive but no full
  component catalog.
- `package-lock.json` (a stray npm artifact) is present but unused; the project
  uses `pnpm-lock.yaml`.
- Docker build requires the `pnpm-lock.yaml` to be committed (now enforced by
  the Dockerfile's `--frozen-lockfile`).
